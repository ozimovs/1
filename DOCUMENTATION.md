# Документация проекта

Обзор всех документов проекта и когда их использовать.

---

## agents.md

**Для кого:** AI-агенты, разработчики.

**Содержание:**
- Тон и правила кода (без внешних библиотек, mobile-first, тёмная тема, зелёный accent).
- **Правила документации:** перед правкой читать существующую документацию, всегда создавать подробную документацию по своему коду.
- Git: коммиты при серьёзных изменениях, минимум раз в 15 минут.
- **Контекст проекта:** DeFi-трекер, ключевые файлы, стек.
- **Правила по проекту:** Redis, расчёт прибыли, фильтры, позиции debankGone, ручные позиции, UI.
- Trigger Rules: при добавлении endpoint — обновлять DEFI_MANUAL_API.md; при изменении схемы Redis — проверять обратную совместимость.

**Когда читать:** перед любыми изменениями в проекте.

---

## SETUP.md

**Для кого:** разработчики при первом запуске.

**Содержание:**
- Создание `.env` и подстановка DeBank API key.
- Локальная разработка через `vercel dev`.
- Деплой на Vercel (переменные окружения).
- Использование прокси вместо прямых запросов к DeBank.

**Когда читать:** при первой настройке окружения или добавлении DeBank API.

---

## KV_SETUP.md

**Для кого:** разработчики при первом деплое с Redis.

**Содержание:**
- Минимальные шаги для подключения Redis в Vercel.
- Storage → Create Database → Upstash Redis.
- Переменные `KV_REST_API_URL`, `KV_REST_API_TOKEN`.

**Когда читать:** при настройке хранилища ручных данных.

---

## DEFI_MANUAL_API.md

**Для кого:** разработчики API и фронтенда.

**Содержание:**
- **POST /api/defi-position-manual:** контракт запроса (openedAt, initialDepositUsd и др.).
- **Redis:** формат ключа и JSON-значения.
- **GET /api/defi-positions:** структура ответа (manualOpenedAt, manualInitialUsd, daysOpen, profitUsd).
- Примечание: APR рассчитывается на фронте, в API не возвращается.
- Таблица Frontend: какие пропы использовать для отображения и отправки.

**Когда читать:** при изменении API ручных позиций или добавлении новых endpoint. Обновлять при каждом изменении контракта API.

---

## DEFI_MANUAL_KV.md

**Для кого:** разработчики, работающие с KV/Redis.

**Содержание:**
- Где хранится (REDIS_URL, Vercel Storage).
- Формат ключа: `wallet:{address}:position:{positionId}`.
- Структура positionId и position_key.
- Примеры JSON-значения в Redis и примера позиции из /api/defi-positions.
- Таблица изменённых файлов (package.json, api/*, lib/redis.js, debank.html, vercel.json).

**Когда читать:** при работе с ручными поправками позиций, миграциях схемы Redis.

---

## .cursor/commands/dis.md

**Для кого:** AI-агенты при запросе `/dis`.

**Содержание:**
- Ссылка на skill webd (web design system).
- Правила: Component-First, Design-First, Conversion-First.
- Рекомендации по UI-библиотекам и workflow.

**Когда читать:** при вызове команды `/dis` для дизайн-подсказок.

---

## Правила обновления документации

| Событие | Действие |
|---------|----------|
| Новый API endpoint | Обновить DEFI_MANUAL_API.md или SETUP.md |
| Изменение схемы Redis | Обновить DEFI_MANUAL_KV.md, проверить обратную совместимость |
| Изменение структуры ответа /api/defi-positions | Обновить DEFI_MANUAL_API.md, DEFI_MANUAL_KV.md |
| Новые правила для агентов | Обновить agents.md |
| Изменение процедуры настройки | Обновить SETUP.md или KV_SETUP.md |
