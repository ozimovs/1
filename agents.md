## Контекст проекта
- **Что это:** DeFi-трекер активов по адресу кошелька. Загрузка данных из DeBank API, хранение ручных данных (дата, сумма, статус) в Redis/Vercel KV.
- **Ключевые файлы:**
  - `debank.html` — основная страница трекера (поиск по адресу, таблица DeFi-позиций, диаграмма активов, модалки).
  - `index.html` — лендинг/главная.
  - `api/defi-positions.js` — получение и объединение позиций (DeBank + Redis).
  - `api/defi-position-manual.js` — сохранение/удаление ручных данных (POST/DELETE).
  - `api/debank-*.js` — прокси к DeBank API.
  - `lib/redis.js` — клиент Redis (Vercel KV).
- **Стек/стиль:** Vercel serverless, Redis (KV), vanilla HTML/CSS/JS. Тёмная тема по умолчанию, зелёный accent (#00e5a0), шрифты Syne и Space Mono.

## Правила по проекту (из опыта работы)

### Redis и ручные данные
- Ключ: `manual:{wallet}:{chain}:{protocolId}:{positionKey}` — не менять (иначе потеряются старые записи).
- При обновлении ручных данных — сначала читать существующий JSON из Redis, merge только пришедшие поля, записывать обратно. Никогда не перезаписывать «с нуля».
- Добавляя новые поля — расширять объект через merge, не удалять старые.

### Расчёт прибыли
- Открытые позиции: `profitUsd = (total_usd + withdrawnUsd) - initialDepositUsd`.
- Закрытые позиции: `profitUsd = withdrawnUsd - initialDepositUsd` (не добавлять total_usd — двойной учёт).

### Фильтры и отображение
- Фильтр по умолчанию — «Открытые» (`lastPositionFilter = 'open'`).
- Позиции &lt; MIN_POSITION_PCT (1%) от портфеля — скрывать в «Прочие позиции», закрытые — всегда показывать отдельно.
- «Прочие позиции» и «Токены на кошельке» — одна агрегированная строка внизу таблицы.

### Позиции, которые DeBank больше не возвращает
- Если позиция была в DeBank, а в ответе её нет — не удалять: показывать из кэша с пометкой `debankGone` (подсветка, подсказка «укажите статус close»).
- Пользователь сам выставляет статус close, позиция остаётся в реестре.

### Ручные позиции (isFullyManual)
- Визуальное отличие: класс `.manual-position`, приглушённый фон.
- Маркер «M» или fallback-иконка (первая буква протокола в `.protocol-placeholder`), если нет protocol_logo_url.
- Кнопка удаления с подтверждением.

### UI
- Без горизонтального скролла: адаптивный layout, card-style на мобильных, collapsible при необходимости.
- Колонка «Состояние» — компактный бейдж (open/close), минимальная ширина.
- Ссылка рядом с протоколом — на сайт протокола, а не DeBank.
- Диаграмма распределения активов — таблица внизу блока, не donut над таблицей.
- Карточка «Открытых протоколов» рядом с «Текущая доходность».
- i18n: RU/EN для всех UI-строк.
